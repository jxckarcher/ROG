### WS Message Types Handled (`store.js:connectWs`)

| Event/Type | Handling |
|---|---|
| `event connect.challenge` | Ed25519 sign, send `connect` req |
| `res hello-ok` | Set wsConnected=true, push system message |
| `res` (ok=true, not hello-ok) | Silently ignored |
| `res` (ok=false) | Push `⚠ errMsg` system bubble |
| `event health/tick/status/channels.status/heartbeat/connect.challenge/doctor.memory.status/logs.tail/presence` | Silently discarded (SILENT_EVENTS set) |
| `event agent` (stream=assistant, delta present) | Append delta to last matching-runId glitch bubble, or create new bubble |
| `event agent` (stream=lifecycle phase=end) | NOT handled → chatSending stays false, no "done" signal |
| `event chat` (state=final) | Find last runId bubble, replace text with canonical final content |
| `event chat` (state=delta) | Silently discarded (using agent stream instead) |

**Known gaps:** `agent lifecycle:end` is silently discarded — this is why typing indicator doesn't work.

### SSH Runner

```js
runCmd(cmd) → invoke('ssh_run', { host, user, cmd })
```

- Bare ssh_run — no token injection
- GitHub panel injects GH_TOKEN via `bashL()` helper:
  ```bash
  T=$(tr "\0" "\n" < /proc/$(pgrep -f openclaw | head -1)/environ | grep -E "^(GH_TOKEN|GITHUB_TOKEN)=")
  export GH_TOKEN="$T"; <cmd>
  ```
- Input escaping is inconsistent:
  - Most places: `replace(/"/g, '\\"')`
  - `appendMemory`: escapes `"` and backticks
  - `safeFsPath`, `safePath`, `safeBranch`: strip unsafe chars
  - **Unsafe:** `saveFile` inlines base64 into a python3 `-c` string without further escaping — safe only because content is pure base64

## 2. What's Working vs Broken

### Chat ✅/⚠
| Feature | Status |
|---|---|
| WS connect + handshake | ✅ Working |
| Basic message send + receive | ✅ Working |
| SSH fallback (no WS) | ✅ Working |
| Streaming rendering | ⚠ **BROKEN** — multi-turn agent responses overwrite same bubble |
| Typing indicator | ⚠ **BROKEN** — chatSending set false immediately after send, dots never show |
| Markdown rendering | ❌ Missing — `<pre>` only, no syntax highlight |
| Context attachments / file send | ❌ Missing |
| Chat history / multiple sessions | ❌ Missing |

**Root cause of streaming bug:** When Glitch uses a tool mid-response, the server resets cumulative `data.text` on the next assistant turn (seq gap). The code appends `data.delta` to the existing same-runId bubble, so "I'll update memory…" becomes "I'll update memory…I" (new turn's delta appended to old turn's text), then "NO" (final tool-use reply), then `chat.final` replaces it all with "NO". **Fix:** detect text regression (`data.text.length < last.text.length`) and start a new bubble.

**Root cause of typing indicator:** `sendChat` calls `sendWs(text)` which returns `true`, then immediately sets `chatSending: false`. The WS response is async. Store never re-sets `chatSending: true` when streaming starts. **Fix:** keep `chatSending: true` until `agent lifecycle:end` or `chat.final` for the active runId.

## 5. Scary Bits

### Secrets
- `GW_TOKEN` is hardcoded in `store.js` line 5 as a JS constant — baked into the built app bundle. This is the gateway auth token. It's client-facing by design (not an API key), but worth moving to env var or Tauri config.
- Device keys (Ed25519 key pair) are in `localStorage` under `glitch-ui:device-keys` — accessible to anyone with DevTools. Low risk since it requires physical access to device.
- PIN hash is in `localStorage` — SHA-256('glitch:' + pin). Brute-forceable offline if localStorage is dumped.

### Accidental Push Risk
- **WorkspacesPanel `commitAndPush()`** runs `git add -A && git commit && git push` with ZERO confirmation. One wrong click while in a repo with unstaged changes pushes everything. Sprint 5 Item 4 fixes this.
- GitHub panel `pullRepo()` could overwrite local VPS changes with `git pull` — no warning if there are uncommitted changes.

### Accidental Money Spending
- Budget gate is client-side only. If the store is reset (page refresh, localStorage cleared), the `_runs` counter resets to 0. Limits can be bypassed by refreshing.
- Overnight mode blocks `chat.send` from the UI but does NOT prevent cron jobs already running on the VPS from spending. Cron jobs run independently of this UI.
- `addOvernightProject()` runs an `openclaw cron add` with a max-tokens flag but OpenClaw enforces this server-side — we don't verify it was respected.
- No WS-side budget enforcement — if Glitch gets into a tool loop, the client won't stop it.

### Unsafe Shell
- `appendMemory(entry)` escapes `"` and backticks but the `glitchlog` script is a shell wrapper — the entry is still passed as a shell argument. Carefully crafted entries with `$(...)` or `;` could still execute if the script uses `eval` internally.
- `saveFile` uses python3 with inline base64 — the path is `safeFsPath` filtered but only removes `..` and non-alnum/dot/dash/slash. A path like `/root/.bashrc` is valid and writable.

## 6. Non-Negotiables (carry into every sprint)

1. **No silent file writes** — diff → review → apply. Never `saveFile(proposed)` without the DiffEditor step.
2. **No auto-clone or auto-push** — clone must be user-triggered button. Push must show diff summary first.
3. **Budget gate must block autonomy when Overnight Mode is on** — currently enforced via `_checkAndRecord(type)`. Do not remove.
4. **No destructive ops without confirm dialog** — delete, rename, rm -rf, cron delete.
